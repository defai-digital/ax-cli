# Product Requirement Documents (PRD)

This directory contains Product Requirement Documents for ax-cli features and improvements.

PRD files follow the naming convention: `YYYY-MM-DD-feature-name.md`

## PRD Template

New PRDs should include:
- Executive Summary
- Problem Statement
- Goals & Success Metrics
- User Stories
- Detailed Requirements
- Technical Design
- Implementation Phases
- Risks & Mitigations
