## เริ่มต้นอย่างรวดเร็ว

เริ่มได้ภายในหนึ่งนาที:

```bash
npm install -g @defai.digital/ax-grok
ax-grok setup
ax-grok
```

**เหมาะสำหรับ:** ค้นหาเว็บแบบสด, วิชัน, การให้เหตุผลขั้นสูง, คอนเท็กซ์ 2M

รัน `/init` ภายใน CLI เพื่อเริ่มต้นบริบทของโปรเจกต์

### โมเดลโลคัล/ออฟไลน์ (ax-cli)

สำหรับการทำงานแบบโลคัลผ่าน Ollama, LMStudio หรือ vLLM ให้ใช้ `ax-cli`:

```bash
npm install -g @defai.digital/ax-cli
ax-cli setup   # กำหนด URL ของเซิร์ฟเวอร์โลคัล
```

ax-cli ทำงานได้กับ **ทุกโมเดล** ที่อยู่บนเซิร์ฟเวอร์โลคัลของคุณ เพียงระบุแท็กโมเดลตอนตั้งค่า (เช่น `qwen3:14b`, `glm4:9b`).

**ตระกูลโมเดลแนะนำ:**

| โมเดล | เหมาะสำหรับ |
|-------|------------|
| **Qwen** | เหมาะที่สุดสำหรับงานเขียนโค้ดโดยรวม |
| **GLM** | รีแฟกเตอร์และเอกสาร |
| **DeepSeek** | ทำงานไว คุณภาพดี |
| **Codestral** | C/C++/Rust และงานระบบ |
| **Llama** | ความเข้ากันได้และ fallback ที่ดีที่สุด |

## การติดตั้ง

### ข้อกำหนด

- Node.js 24.0.0+
- macOS 14+, Windows 11+ หรือ Ubuntu 24.04+

### ติดตั้ง

```bash
npm install -g @defai.digital/ax-grok   # Grok (xAI)
```

### ตั้งค่า

```bash
ax-grok setup
```

ตัวช่วยตั้งค่าจะพาคุณทำ:
1. เข้ารหัสและเก็บ API key อย่างปลอดภัย (ใช้ AES-256-GCM)
2. กำหนดโมเดล AI เริ่มต้นและค่าที่ต้องการอื่นๆ
3. ตรวจสอบการตั้งค่าเพื่อให้แน่ใจว่าทุกอย่างถูกต้อง

## การใช้งาน

### โหมดโต้ตอบ

```bash
ax-grok              # เริ่มเซสชัน CLI แบบโต้ตอบ
ax-grok --continue   # ต่อบทสนทนาเดิม
ax-grok -c           # แบบย่อ
```

### โหมด headless

```bash
ax-grok -p "วิเคราะห์โค้ดเบสนี้"
ax-grok -p "แก้ข้อผิดพลาด TypeScript" -d /path/to/project
```

### แฟล็กพฤติกรรมเอเจนต์

```bash
# เปิดโหมด ReAct (คิด → ลงมือทำ → สังเกต)
ax-grok --react

# เปิดการตรวจสอบ TypeScript หลังเฟสการวางแผน
ax-grok --verify

# ปิดการแก้ตัวเองเมื่อผิดพลาด
ax-grok --no-correction
```

ค่าเริ่มต้นคือเปิดการแก้ตัวเอง (เอเจนต์จะลองใหม่พร้อมการสะท้อนความคิด). ReAct และการตรวจสอบถูกปิดไว้ แต่สามารถเปิดเพื่อให้การให้เหตุผลเป็นระบบมากขึ้นและตรวจคุณภาพได้.

## การเริ่มต้นโปรเจกต์

คำสั่ง `/init` จะสร้างไฟล์ `AX.md` ที่รากโปรเจกต์ ซึ่งเป็นไฟล์บริบทแบบครบถ้วนที่ช่วยให้ AI เข้าใจโค้ดเบสของคุณ

### การใช้งานพื้นฐาน

```bash
ax-grok
> /init                    # วิเคราะห์มาตรฐาน (แนะนำ)
> /init --depth=basic      # สแกนเร็วสำหรับโปรเจกต์เล็ก
> /init --depth=full       # วิเคราะห์ลึกพร้อมแม็พสถาปัตยกรรม
> /init --depth=security   # รวมการตรวจสอบความปลอดภัย (secrets, API อันตราย)
```

### ระดับความลึก

| ระดับ | วิเคราะห์อะไร | เหมาะสำหรับ |
|-------|--------------|------------|
| `basic` | ชื่อ, ภาษา, tech stack, สคริปต์ | ตั้งค่าเร็ว, โปรเจกต์เล็ก |
| `standard` | + สถิติโค้ด, วิเคราะห์เทสต์, เอกสาร | โปรเจกต์ส่วนใหญ่ (ค่าเริ่มต้น) |
| `full` | + สถาปัตยกรรม, dependencies, hotspots, คู่มือใช้งาน | โค้ดเบสขนาดใหญ่ |
| `security` | + สแกน secrets, ตรวจจับ API อันตราย, รูปแบบ auth | โปรเจกต์ที่เน้นความปลอดภัย |

### เอาต์พุตแบบปรับตามโปรเจกต์

คำสั่ง `/init` จะปรับความละเอียดของเอาต์พุตตามความซับซ้อนของโปรเจกต์โดยอัตโนมัติ:

| ขนาดโปรเจกต์ | จำนวนไฟล์ | เอาต์พุตโดยทั่วไป |
|--------------|----------|-------------------|
| เล็ก | <50 ไฟล์ | กระชับ เฉพาะสาระสำคัญ |
| กลาง | 50-200 ไฟล์ | เอกสารมาตรฐาน |
| ใหญ่ | 200-500 ไฟล์ | รายละเอียดพร้อมบันทึกสถาปัตยกรรม |
| Enterprise | 500+ ไฟล์ | ครอบคลุมทุกส่วน |

### ตัวเลือก

| ตัวเลือก | รายละเอียด |
|---------|------------|
| `--depth=<level>` | ตั้งค่าความลึกการวิเคราะห์ (basic, standard, full, security) |
| `--refresh` | อัปเดต AX.md เดิมด้วยการวิเคราะห์ล่าสุด |
| `--force` | สร้างใหม่แม้มี AX.md อยู่แล้ว |

### ไฟล์ที่ถูกสร้าง

| ไฟล์ | วัตถุประสงค์ |
|------|--------------|
| `AX.md` | ไฟล์บริบทหลักของ AI (สร้างเสมอ) |
| `.ax/analysis.json` | ข้อมูลการวิเคราะห์เชิงลึก (เฉพาะ full/security) |

### การทำงานของการฉีดบริบท

เมื่อเริ่มบทสนทนา AX CLI จะอ่าน `AX.md` และฉีดเข้าไปในหน้าต่างบริบทของ AI โดยอัตโนมัติ ซึ่งหมายความว่า:

1. **AI รู้จักโปรเจกต์ของคุณ** - คำสั่ง build, tech stack, กฎ/รูปแบบ
2. **ไม่ต้องอธิบายซ้ำ** - AI จำโครงสร้างโปรเจกต์ได้
3. **แนะนำโค้ดได้ดีขึ้น** - ทำตามรูปแบบและกฎที่มีอยู่

```
You run: ax-grok
         ↓
System reads: AX.md from project root
         ↓
AI receives: <project-context source="AX.md">
             # Your Project
             ## Build Commands
             pnpm build
             ...
             </project-context>
         ↓
AI understands your project before you ask anything!
```

**ลำดับความสำคัญ** (เมื่อมีไฟล์บริบทหลายไฟล์):
1. `AX.md` (แนะนำ) - รูปแบบไฟล์เดี่ยวใหม่
2. `ax.summary.json` (legacy) - สรุป JSON
3. `ax.index.json` (legacy) - ดัชนี JSON แบบเต็ม

### การย้ายจากรูปแบบเดิม

หากมีไฟล์ legacy (`.ax-grok/CUSTOM.md`, `ax.index.json`, `ax.summary.json`) ให้รัน:

```bash
> /init --force
```

คำสั่งนี้จะสร้างรูปแบบไฟล์เดี่ยวใหม่ `AX.md` จากนั้นสามารถลบไฟล์ legacy ได้

## การผสานรวม MCP

ขยายความสามารถด้วย [Model Context Protocol (MCP)](https://modelcontextprotocol.io) — มาตรฐานเปิดสำหรับเชื่อมต่อผู้ช่วย AI กับเครื่องมือภายนอก, API และแหล่งข้อมูล:

```bash
ax-grok mcp add figma --template
ax-grok mcp add github --template
ax-grok mcp list
```

**เทมเพลตที่มีให้ใช้:** Figma, GitHub, Vercel, Puppeteer, Storybook, Sentry, Jira, Confluence, Slack, Google Drive และอื่นๆ

## การผสานรวม AutomatosX

AX CLI ผสานรวมกับ [AutomatosX](https://github.com/defai-digital/automatosx) ซึ่งเป็นระบบ AI แบบหลายเอเจนต์ที่มีการแก้บั๊กอัตโนมัติ การรีแฟกเตอร์อัจฉริยะ และเอเจนต์เฉพาะทางมากกว่า 20 รายการ

ในโหมดโต้ตอบ (`ax-grok`) แค่ถามตามธรรมชาติ:

```
> โปรดสแกนและแก้บั๊กในโค้ดเบสนี้

> รีแฟกเตอร์โมดูลยืนยันตัวตน โดยโฟกัสการลบโค้ดที่ตายแล้ว

> ใช้เอเจนต์ความปลอดภัยเพื่อตรวจสอบ API endpoints

> รีวิว PRD นี้และทำงานร่วมกับเอเจนต์โปรดักต์เพื่อปรับปรุง

> ขอให้เอเจนต์แบ็กเอนด์และฟรอนต์เอนด์ช่วยกันทำฟีเจอร์ลงทะเบียนผู้ใช้
```

**สิ่งที่ได้รับ:**
- **การแก้บั๊ก**: ตรวจจับการรั่วของไทเมอร์, การล้างทรัพยากรที่ขาด, ปัญหาทรัพยากร - แก้ไขอัตโนมัติพร้อมความปลอดภัยแบบ rollback
- **การรีแฟกเตอร์**: ลบโค้ดที่ตายแล้ว, แก้ความปลอดภัยของชนิดข้อมูล, ลดความซับซ้อน - ตรวจสอบด้วย typecheck
- **เอเจนต์ 20+ ตัว**: แบ็กเอนด์, ฟรอนต์เอนด์, ความปลอดภัย, สถาปัตยกรรม, DevOps, ข้อมูล และอื่นๆ

ดู [AutomatosX Guide](docs/AutomatosX-Integration.md) สำหรับรายชื่อเอเจนต์ ตัวเลือกขั้นสูง และการตั้งค่า

## หน่วยความจำโครงการ

ลดค่าโทเค็นและปรับปรุงการจดจำบริบทด้วยแคชอัจฉริยะที่เก็บและดึงข้อมูลโครงการที่เกี่ยวข้อง เพื่อลดการประมวลผลซ้ำ.

```bash
ax-grok memory warmup    # สร้างแคชบริบท
ax-grok memory status    # ดูการกระจายโทเค็น
```

## ความปลอดภัย

- **การเข้ารหัส API key:** AES-256-GCM พร้อม PBKDF2 (600K รอบ)
- **ไม่มีเทเลเมทรี:** ไม่เก็บข้อมูล
- **การป้องกันแบบ CVSS:** ป้องกันช่องโหว่ทั่วไป เช่น Command Injection (CVSS 9.8), Path Traversal (CVSS 8.6) และ SSRF (CVSS 7.5)
